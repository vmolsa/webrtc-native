Interface RTCRtpSynchronizationSource

The RTCRtpSynchronizationSource dictionary contains information about a given synchronization source (SSRC), including the most recent time a packet that the source contributed to was played out. The browser must keep information from RTP packets received in the previous 10 seconds.

Optional audioLevel

audioLevel: number
  • Defined in ortc.ts:2468

The audio level contained in the last RTP packet played from this source. audioLevel will be the level value defined in [RFC6464], if the RFC 6464 header extension is present.

source: number

The SSRC identifier of the synchronization source.

Optional timestamp

timestamp: number
  • Defined in ortc.ts:2455

The timestamp of type DOMHighResTimeStamp [HIGHRES-TIME], indicating the most recent time of playout of an RTP packet from the source. The timestamp is defined in [HIGHRES-TIME] and corresponds to a local clock.

voiceActivityFlag

voiceActivityFlag: boolean
  • Defined in ortc.ts:2478

Whether the last RTP packet received from this source contains voice activity (true) or not (false). Since the "V" bit is supported in [RFC6464] but not [RFC6465], the voiceActivityFlag attribute will only be set when receivers enable the client-mixer header extension (setting the vad attribute to true), and when RTP packets are received from senders enabling the client-mixer header extension (setting the vad attribute to true).
